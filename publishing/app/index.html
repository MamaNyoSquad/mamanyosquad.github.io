Venus Island Diaries

Release Notes - v0.3.6-prerelease


Main Release Notes


Our approach to migrating our Inside Stories

We had our first Inside Story -- Deep Arrogance -- migrated to the website due to the shutdown of Revue. This has been improved, alongside the addition of existing Inside Stories into the migration phase.

Our primary focus on this approach is to make it mobile-friendly since we'll be including them in the app.

All contents are taken from their original Google Docs files, along with some minor spelling, grammar and formatting adjustments. We've also included effects not present in the Docs for a more immersive experience.

Desktop and Web users, you can still view the documents as they would on mobile via https://mamanyosquad.github.io/publishing.


Improvements to our ImageView implementation

We've done a bit of research on how we can implement a better way to display our images without sacrificing the APK size. We came across an article by GeeksforGeeks about how we can appeal to such implementation, and safely delete the image resources in our project.

Albeit a bit slow to load, the ImageView components rendered successfully!

For those who want to take a look at the implementations that we base on, head over to the Releases page via the FAB on the bottom-right and look for v0.3.6-prerelease on the release title.


Some adjustments to build.gradle.kts

We've done a bit of coding work to be able to prepare ourselves whenever we finally push the official v1.0-release version of the app! For transparency, we provided the following code as shown below: 

        buildTypes {
          named("release") {
            isMinifyEnabled = true
            isShrinkResources = true
            setProguardFiles(listOf(getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro"))
            versionNameSuffix = "-release"
          }
          named("debug") {
            signingConfig = signingConfigs.getByName("debug")
            versionNameSuffix = "-prerelease"
          }
        }

Note that the new versioning implementation is denoted in bold text in the pre block.

Moving forward with this approach, all future -prereleases will be published under the "debug" option, while -release will be under the "release" option.

For the full code, head over to vidiaries-app/app/build.gradle.kts under the development branch, with more details under the hood. Keep on reading for future implementation notes.



Other Release Notes


Multiple Kotlin files, one WebView

As we continue to expand our migration phase in the middle of development from v0.3.5-prerelease to v0.3.6-prerelease, we tried many ways to get pages going on one WebView. The best implementation that we can proceed with is to invoke multiple Kotlin files bearing the respective Inside Story releases in each file, and direct the said files to a WebView layout XML.

If you want to take a look at the implementations that we did, head over to vidiaries-app/app/src/main/java/mamanyosquad/vidiaries directory under the development branch.


Making site-wide changes

As part of the redesign efforts, we'll begin improving the publishing page on a regular basis. This release note covers the following adjustments:

  • Changes to our FAB (Floating Action Button), and optimizing it against unnecessary attributes
    • This also affects the publishing page, where it employs the same FAB implementation
  • .fillWidth blocks are more blog-like, similar to the Inside Story pages
  • Inside Story pages have splash screens implemented
    • For more information, visit thebelovedmoon.github.io's sourcecode on GitHub


Future Implementations


Adjustments to build.gradle.kts

We're working on optimizing the experience without sacrificing the APK size even more! In a future pre-release, all implementations that we employed on our "release" option will carry over to "debug".


Tablet view

Once we're done with all of the optimizations as we work on more adjustments in the app, we'll make the app tablet-friendly by invoking the Tablet View Layout XML, usable for larger screens and foldables.
